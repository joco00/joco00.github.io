My brother says simple isn't easy. Paul Graham has written about the importance of Simplicity. (There are definitly more examples of people talking about simplicity) There is a quote (i cant find who said it) If you cant say it simply do you understand it? In software you see this idea all the time, confusion is abstracted out into a simpler language or interface. 0/1's -> machine code -> assembly code -> itermidiary code -> high level code The bottom (left) if very complex, the top (right) is also very complex. But if you were trying to do the right with the left it would be impossible. Simplicity =/= Easy
